# Decisions: omp-model-metadata-defaults-m4k

| # | Date | Decision | Rationale | Confidence |
|---|------|----------|-----------|------------|
| 1 | 2026-06-17 | Set maxTokens via patch.json, not by editing models.json | models.json is auto-generated from the Makora API and would be overwritten by `update-models.js`. patch.json is the manual override layer. | High |
| 2 | 2026-06-17 | Set update-models.js default to 8192, not 0 | 0 means "unlimited" which is dangerous for reasoning models. 8192 is a safe universal default that patch.json can override per-model. | High |
| 3 | 2026-06-17 | DeepSeek V4 models: maxTokens = 32768 | DeepSeek API supports `max_tokens` up to 32768. These are the highest-capability models and users expect full output range. | Medium |
| 4 | 2026-06-17 | Reasoning models: maxTokens = 16384 | Reasoning models need extra token budget for thinking. 16384 provides ample room (e.g., 8K thinking + 8K output) without being unlimited. | Medium |
| 5 | 2026-06-17 | Non-reasoning Llama 3.3: maxTokens = 8192 | Meta's recommended default for Llama 3.3. Non-reasoning models don't need the extra thinking budget. | High |
| 6 | 2026-06-17 | vision.maxImagesPerRequest = 5 for all multimodal models | Standard vLLM default. 5 images is generous for coding workflows (screenshots, diagrams). Can be increased later if needed. | Medium |
| 7 | 2026-06-17 | Hardcode maxTokens per-model in patch.json rather than compute from contextWindow | No reliable formula exists (contextWindow / N). Model-specific research produces better values. | High |
| 8 | 2026-06-17 | Don't touch cost fields | Makora doesn't expose pricing. Setting fake costs is misleading. Leave at 0 until Makora provides pricing. | High |
| 9 | 2026-06-17 | Don't change models.json directly | models.json is regenerated by `node scripts/update-models.js` and should only be edited via that script. The script's default change is sufficient. | High |
